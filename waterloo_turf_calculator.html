Waterloo Turf Job Calculator
Complete User Guide
What this tool does
The Waterloo Turf Job Calculator helps you estimate material quantities and build customer quote options for artificial turf installation jobs. It handles turf ordering, infill, rock/base materials, edging, and miscellaneous items — then automatically generates all quote combinations so you can present pricing options to your customers.
Understanding Field Colors
Green — Auto-calculated. Computed from your inputs. Read-only.
Yellow — Manual input. Enter your own value here.
White — Standard input. General editable field.
Getting Started: Settings First

Before creating any projects, go to the Settings tab and configure your catalogs. This data pre-fills your projects automatically.

Turf Products

Add each turf product you sell. Set the Type (Standard or Putting Green), Vendor Product Name (how it appears on supplier orders), and Cost per SqFt (used in quote totals). Rolls are always 15 ft wide.

Infill Products

Add your infill products (e.g. PFS Silica Sand, GD Medium Sand, GD Putting Sand). Set Lbs per SqFt so bag quantities calculate automatically, and Cost per Bag so infill pricing appears in quotes.

Rock / Base Materials

Add your base materials (e.g. 1/4" x 3/4" Clean Crush at 4" depth, 1/4" Minus at 1" depth). Set the Default Depth — this is locked to what you define here and cannot be changed per-project. Price per SqFt @ 1" and Cost per Ton are for your own reference when ordering; they don't appear in customer quotes (rock cost is built into your per-sqft labor rate instead — see Labor Rates below).

Miscellaneous Items

Add any other materials you regularly include (seam tape, adhesive, nails, staples, etc.). Set the name, unit, and price. These can be added to individual quotes later.

Labor Rates & Price Sheet

Define your crew rate sets. You can have multiple crews with different rates. Each project can be assigned a specific crew using the Crew dropdown in the topbar. All rates start blank — nothing is pre-filled, so set each one for your business before relying on quote totals. Key rates that drive quote totals:

  • Standard Turf Install — per sqft rate for standard yard install, applied to the portion of the yard outside any putting green
  • Putting Green Install — per sqft rate for putting green area
  • Bender Board Edging Install — per linear ft install rate, shown dynamically based on your active crew's rate
  • Bender Board Materials — price per 20 ft board, including stakes & screws
Creating a Project

Click ＋ New Project in the sidebar. Fill in the project name, then optionally import a Moasure CSV right from the creation modal — doing so pre-fills the Installed SqFt for all turf rows you select, and attaches the layout so the Layout tab is ready as soon as the project opens.

Check off turf products, then enter Installed SqFt for each. Sqft to Order is optional at creation — leave it blank if you don't have it yet (e.g. you haven't measured/imported the layout). "Create Project" only requires at least one turf product checked. You can fill in or calculate Sqft to Order afterward, either directly on the Quote Builder tab or via the Layout tab's roll planner once you import a Moasure CSV. Set the Role for each turf row (Base Yard, Alt Turf Option, Putting Green) at creation time rather than after. Check infill products, enter edging and putting green sqft if known. Rock rows are auto-created from your Settings catalog.

Projects save automatically and appear in the sidebar sorted by name, newest, or oldest.

Quote Builder Tab

This is the main working area for a job. It has five sections:

1. Turf

Each row represents a turf product on this job. Enter:

  • Installed SqFt — the actual area being installed. Drives labor, infill, and rock calculations.
  • SqFt to Order — the sqft you plan to order, accounting for waste and yard shape. This will typically be higher than installed sqft. Optional at project creation — enter manually here, or use the Layout tab to calculate it from the Moasure CSV. Until filled in, Linear Ft, Ordered SqFt, and turf material cost in the quote show as blank/$0 rather than a placeholder value.
  • Linear Ft ↑ — auto-calculated: ceil(SqFt to Order ÷ 15). Rounds up to the next full roll length.
  • Ordered SqFt ↑ — auto-calculated: Linear Ft × 15. The actual sqft purchased after rounding up to a full roll width. Used for turf material cost in quotes.
  • Role — controls how this row appears in quote combinations. Changing a row's role immediately recalculates rock and infill quantities.
Base Yard — the standard turf product. Always included in quote totals. Infill and rock calculations are based on this sqft.
Alt Turf Option — an alternative turf product (e.g. a premium upgrade). Same install area as the base yard but a different product and price. Creates separate quote option cards so the customer can compare.
Putting Green — a putting green area. If the customer chooses this, its sqft is subtracted from the base yard for crew pay purposes (since putting green pays at a higher rate). Creates separate quote combos with and without the putting green.
2. Infill

Each row is an infill product. SqFt auto-populates from turf installed sqft based on the Tier — Standard and Upgraded infill get the base yard sqft, Putting Green infill gets the putting green sqft. You can override any sqft manually.

Enter Lbs per SqFt (how dense the infill is) and Cost per Bag (pre-filled from Settings if set). Bags ↑ and Line Cost ↑ calculate automatically (50 lbs per bag).

The Tier dropdown controls how this infill appears in quote options:

Standard — the base infill option. Included in all turf scenarios.
Upgraded — a premium alternative. Creates a parallel set of quote cards so customers can see Standard vs Upgraded infill pricing side by side.
Putting Green — infill for the putting green area only (e.g. GD Putting Sand). Only included in scenarios that have a putting green option. SqFt auto-fills from the putting green turf row.

Tip: Use the ↻ Refresh from SqFt button to re-sync infill sqft if you change turf quantities.

3. Edging (Bender Board)

Enter the total linear feet of edging. The tool automatically calculates: boards to order (rounded up to 20 ft boards), install labor cost, materials cost, and the total. Rates come from your active crew in Settings, and the per-ft / per-board rates shown next to each field always reflect that crew's actual numbers — if you switch crews or update rates, the labels update too.

4. Miscellaneous Items

Click ＋ Add Item to open a picker showing everything from your Settings → Miscellaneous Items catalog. You can also enter a one-off custom item inline. Set the Qty and the line total calculates automatically (Price/Unit × Qty). The Role dropdown controls which scenarios include this item's cost:

  • Base — included in every quote scenario
  • Putting Green — only added to scenarios that include a putting green option
  • Alt Option — excluded from totals (visible for reference but not priced in)
5. Quote Options

This section auto-generates every meaningful pricing combination from your turf roles, infill tiers, and putting green options. No manual work needed — just fill in the rows above and the quotes build themselves.

How scenarios are built:

  • Each turf combo (Base Yard, plus one card per Alt Turf Option) is combined with each infill tier (Standard, Upgraded) to form a group
  • Within each group, putting green variants appear side by side: one card without a putting green, one per Putting Green turf row

Example — Standard turf + 1 Alt turf, Standard + Upgraded infill, 1 putting green option → 4 groups, 2 cards each = 8 total quote options.

What's included in each total: Labor (standard yard rate + putting green rate if applicable), turf material cost (ordered sqft × cost/sqft from Settings), edging, infill materials (bags × cost/bag), putting green fringe (if configured in the Layout tab — see Putting Green Fringe below), and miscellaneous items — all split correctly by role/tier per scenario.

Layout Tab: Site Layout & Roll Plan

Import the CSV exported from a Moasure measurement to visualize the yard shape and plan how turf rolls lay across it. You can import at project creation time (pre-fills measurements) or later via the Layout tab.

Importing

Click "↑ Import CSV" — either in the New Project modal (pre-fills Installed SqFt for all turf rows) or in the Layout tab after project creation. The shape is drawn on the canvas with roll strips overlaid immediately. "Installed Area" shows the polygon-computed area; use "Apply Area" to push it into a turf row's Installed SqFt.

Multi-Layer CSVs (Layers)

Moasure files can contain more than one measured layer — for example, the main yard outline plus a separate shape for a planter bed, tree well, or putting green cutout inside it. When you import a CSV with multiple layers, the largest shape becomes the primary layer (the main boundary used for roll calculations), and any other shapes appear below it under "Layers" on the right.

Every layer — including the primary one — has a checkbox. Uncheck it to hide that layer on the canvas; this is purely visual and doesn't affect any calculations. This is useful for decluttering the view when a file has several overlapping shapes, or for focusing on one layer while editing.

Each additional (non-primary) layer also has a dropdown that controls how it affects Installed Area:

  • Exclude from Installed Area (hole/cutout) — default. Subtracts that shape's area from the Installed Area total, and draws it on the canvas as a hatched red region — turf doesn't go there.
  • Ignore (informational only) — has no effect on Installed Area. Drawn as a dashed blue outline for reference (e.g. you want to see where a feature is, but it doesn't change how much turf you order).
  • Putting Green (used for fringe calculation) — subtracts that shape's area from Installed Area like Exclude, AND marks it as the putting green outline for the Fringe feature (see below). Drawn as a solid green shape. Only one layer can be the putting green at a time — selecting it on a different layer demotes the previous one back to Exclude.

"Installed Area" shows the adjusted total (after exclusions), with the pre-exclusion total in parentheses if anything was excluded. "Apply Area" and the roll plan's scrap calculation both use the adjusted total. Note that hiding a layer (visibility checkbox) is independent from its exclude/ignore/putting-green mode — a hidden layer can still be excluded from the area total, and a visible layer can be set to "ignore."

Putting Green Fringe

Once a layer is set to Putting Green, a "Putting Green Fringe" section appears below the Layers list. Check "Add a fringe border around the putting green" to enable it, then choose a Fringe Turf Product from your catalog and set the Fringe Width (in feet) — the border strip's width all the way around the green.

The fringe is calculated as a ring around the putting green's actual perimeter (traced from its Moasure outline), offset outward by the fringe width with properly mitered corners — adjacent pieces meet exactly edge-to-edge with no overlap and no gap, so the canvas doesn't show doubled-up or duplicated-looking pieces. Rather than cutting one piece per original boundary point — which for a Moasure-captured outline with curves or rounded corners could mean dozens (or with very fine-grained traces, over a hundred) of tiny pieces — nearby straight or gently-curved sections of the perimeter are first merged into single longer pieces wherever the boundary stays within about half the fringe width of a straight line between their endpoints, capped at Max Roll Length so each piece can still be cut from one roll. Each merged piece is then a simple trapezoid (a rectangle once cut) running between mitered corners; on any unusually sharp or jagged corner where a true miter would spike out unreasonably far, that corner is squared off (beveled) instead, keeping every piece a sensible size. This intentionally orders a bit more material than the bare ring area — the summary shows the ring area, total sqft to order, and the difference between them — in exchange for far fewer seams and cuts, which is worth it for most jobs. The fringe width directly controls all of this: a wider fringe both adds more sqft per piece and allows more merging (since the deviation tolerance scales with width), so changing it reflows the whole plan.

On the canvas, the fringe is drawn in orange around the green. By default each piece is filled and labeled ("Fringe 1", "Fringe 2", etc.) so you can match the diagram to your cut list. Uncheck "Show fringe pieces on canvas" to instead draw a single smooth outline at the fringe width, following the green's actual outline point-for-point — this is closer to how the fringe would actually look once installed, rather than the coarser straight-edged pieces used for cutting/ordering. This only affects the canvas drawing; the Piece List, sqft, and pricing are unaffected either way.

Fringe material cost is added to any quote option that includes a putting green (it has no effect on "No Putting Green" options), using the same turf material pricing as the main yard's products.

Moving & Rotating Layers

Click "✋ Move Layers" to enter move mode. Click and drag any visible layer — the main shape outline or any additional shape — to reposition it on the canvas. This is purely visual: it doesn't change area, roll counts, scrap, or any other calculation, only where things are drawn. Each layer has its own independent position. Click "✓ Done Moving" to exit. Move Layers mode is mutually exclusive with Edit Shape mode.

Each additional (non-primary) layer also has a Rotate slider (0-359°) in the Layers list. This rotates that shape in place about its own center — useful when Moasure captured a shape (e.g. a putting green outline) at the wrong orientation and you need to spin it to match how it actually sits in the yard. Combine with dragging to both rotate and position the shape correctly. Rotation, like position, is purely visual and doesn't affect Installed Area, scrap, or roll calculations.

A layer that's been moved and/or rotated shows a "↺ Reset position" button in the Layers list to snap it back to its original (imported) position and orientation.

Manual Cuts (Butt Seams)

Click "✂ Cut Mode" to enter cut mode. Click anywhere along a roll to add a butt-seam cut at that point — this splits the roll into separate pieces, each with its own cutting margin, labeled "Roll N.1", "Roll N.2", etc. with a red seam line marking the cut. You can add multiple cuts to the same roll to create 3 or more pieces. Click an existing cut line again to remove it. Click "✓ Done Cutting" to exit — Cut Mode is mutually exclusive with Edit Shape and Move Layers modes.

A "Manual Cuts (Butt Seams)" list appears in Roll Results whenever any roll has cuts, showing each cut's position with a "✕" to remove it individually, or a "Clear all cuts on this roll" button — a precise alternative to clicking the (sometimes thin) seam line on canvas. If roll settings change such that a saved cut position no longer falls within that roll's current length, it's shown as "cuts not currently applied" with a button to clear it — so it never becomes invisible/stuck.

Each piece behaves like its own roll for nesting purposes: with "Show purchased roll rectangles" on (and Cut Mode off), drag any piece onto another roll's (or another piece's) hatched waste area to nest it there — exactly like whole-roll nesting. This is useful for splitting off a small leftover piece and tucking it into another roll's offcut instead of ordering it separately.

Cut positions are stored as a distance along the roll's length, so like nesting placements, they're tied to that roll's band position and persist across saves/reloads. If a major roll-setting change (direction, seam offset, width) causes that band to no longer exist, the cuts for it are simply dropped.

Canvas Layout

The canvas and the fields panel sit side-by-side — canvas on the left (fills all available width), fields on the right. The canvas is sized automatically to exactly fit the shape's aspect ratio with no dead whitespace, and adapts as the window resizes. On mobile (≤860px) the fields stack below the canvas.

Zoom

The canvas always opens at the natural fit — no dead space, shape fills the available area. Use ＋ / － to zoom in/out (the canvas grows and scrolls within its container), and ⊙ Fit to snap back to the fitted view. Ctrl+scroll / two-finger trackpad pinch also zooms. All edit and drag interactions remain accurate at any zoom level.

Editing the Shape

Click "✎ Edit Shape" to clean up the imported outline after import:

  • Drag a green point to move it — area and roll layout recompute live.
  • Double-click a point to delete it (minimum 3 points).
  • Double-click on an edge (between points) to insert a new point there.

"↶ Undo" steps back through your last 20 shape edits. Click "✓ Done Editing" to exit. Everything saves automatically. To start over from the original file, re-import the CSV — this replaces the shape and clears edit history.

Shape Rotation (view)

Rotates the drawing to match how you're viewing the yard on-site. Purely cosmetic — does not affect area, roll counts, or any calculations.

Roll Settings

Collapsible section. Parameters that drive the roll plan:

  • Roll Width — width of the turf roll as purchased.
  • Roll Length — maximum length of a single piece; longer runs are split, each adding a Cutting Margin.
  • Side Trimming — width trimmed off each roll for S-seams (Roll Width − Side Trimming = usable width per strip).
  • Cutting Margin — extra length added per cut/seam.
  • Roll Direction — angle of rolls relative to shape. "Horizontal"/"Vertical" snap to 0°/90°. "Auto-minimize waste" searches all 180° × 8 translation offsets and picks the combination with the lowest ordered sqft.
  • Seam Translation — shifts where seam lines fall; can reduce strip count. Max value equals the effective roll width.

Each roll strip only orders the length it actually needs based on where it intersects the shape — strips don't all buy the full bounding-box length. Ordered length is rounded up to the next whole foot, since turf is sold in Roll Width × 1ft increments. This produces accurate per-strip quantities and more realistic scrap percentages.

Roll Results

Shows roll/piece count, Ordered SqFt (what you'd purchase, rounded to whole-foot increments), Usable SqFt (after side trim), Linear Ft, and Scrap (Ordered minus shape area, with %). "Apply Sqft to Order" pushes Ordered SqFt directly into a turf row.

The roll/piece count shows just "N rolls" when the number of rolls to purchase equals the number of installed pieces — the normal case. It shows "N rolls / M pieces" only when these differ.

Pieces = total separately-installed pieces across the whole job: one per band/strip normally, more if a strip is manually cut into butt-seamed sections, or if a single strip is longer than Max Roll Length (which forces an inherent seam even without a manual cut).

Rolls = a global count based on total linear footage: ceil(sum of every piece's ordered length ÷ Max Roll Length). All pieces run the same direction at the same Roll Width, so pieces from different bands/strips can be cut from the same physical roll as long as their combined length fits. For example, 3 separate pieces totaling 75 linear ft all come from a single 100ft roll — shown as "1 roll / 3 pieces" — while 3 pieces totaling 180ft need "2 rolls / 3 pieces".

Toggle "Show purchased roll rectangles" to overlay each roll's full purchased rectangle with the unused (waste) portion hatched in red. This enables drag-and-drop nesting.

Piece List (Length × Width)

A table listing every individual piece that will be cut for this job — each row shows its Roll N / Piece M label, Length (its ordered length along the roll), Width (the roll width), and resulting SqFt. This is the cut sheet for your installers: it tells them exactly how long to cut each piece from the roll, and the total linear footage at the bottom helps you sanity-check how many rolls to bring and plan seam placement to minimize butt seams. Pieces created by Manual Cuts or that have been nested into another roll's waste show a note (e.g. "cut from Roll N / Piece M waste"). If a Putting Green Fringe is configured, the fringe pieces ("Fringe 1", "Fringe 2", ...) are listed too, with their own length/width/sqft.

Drag-and-Drop Nesting

With "Show purchased roll rectangles" on, drag a roll's colored shape (the installed piece) onto another roll's hatched waste area to indicate it can be cut from that roll's offcut rather than ordered separately. Valid drop targets highlight with a green dashed border while dragging. On a successful drop the piece is redrawn relocated into the target roll's waste area, outlined in orange and labeled "from Roll N / Piece M waste," and its area is removed from Ordered SqFt. Drag it back off the waste area to reset. Only works in non-edit mode.

Placements are tied to each roll's position in the shape, so they persist across saves/reloads. If you later change roll direction, seam offset, roll width, or side trim enough that the rolls reflow into different bands, a placement tied to a band that no longer exists is simply dropped — it won't jump onto an unrelated roll. Re-check nesting after making significant roll-setting changes.

Known Limitations

Nesting feasibility is area-based only — the tool checks that a piece's area fits within another roll's waste area, but does not verify geometric fit (shape, orientation). For simple rectangular pieces this is reliable; for irregular shapes, confirm visually before finalizing. Roll strips currently don't split further within a strip for very irregular shapes — treat scrap % as a useful planning number, not a guarantee.

Materials Tab

Shows the material quantities you need to order. Two sections:

Rock / Base

Shows the tons to order for each rock/base material in your catalog, auto-calculated from this job's base turf sqft and each material's depth/density settings. SqFt basis, depth, and pricing are configured once under Settings → Rock / Base Materials rather than per-project — only the resulting tons total is shown here. Remove a row if this job doesn't need that material.

Materials Summary

A full summary of all materials across turf, infill, edging, and rock for easy reference when ordering.

Multiple Crew Rate Sets

Go to Settings → Labor Rates. Click ＋ Add Crew to create a new rate set (e.g. "Main Crew", "Temp Crew"). You can copy rates from an existing crew as a starting point. Edit any rate inline and it saves automatically.

Per-project crew selection: Each project can use a different crew. Use the Crew dropdown in the topbar (next to the project name) to assign a specific crew to the current project — all quote calculations update immediately to use that crew's rates. This lets you quote jobs with different crews without changing the global setting.

Reordering line items: In the Labor Rates table, drag any row by its ⠿ handle to reorder the line items within a crew.

Backup, Sync & Restore

Your data is stored in your browser's local storage. This means it can be lost if you clear your browser cache. Use the ⇄ Sync / Backup menu at the bottom of the sidebar regularly to export a JSON file. Keep backups somewhere safe (email to yourself, save to Google Drive, etc.).

Selecting Projects

Each project in the sidebar list has a checkbox. Check "All" at the top of the project list to select everything, or check individual projects. The Sync menu shows how many are currently selected.

Export Options
  • Export Selected — saves a JSON file containing only the checked projects. Does not include catalog, crew rates, or misc-item settings — use this for moving a specific job to another device (e.g. syncing one project to your iPad) without overwriting that device's settings.
  • Export Everything (Projects + Settings) — saves all projects plus your full catalog, crew rates, and misc items. This is the complete backup — use it for disaster recovery, or to seed a brand-new install with your catalog/crew setup (works even with zero projects selected — the catalog/settings export regardless).
Import Options
  • Import & Merge — adds projects from the file to your existing projects. If a project ID in the file already exists on this device, you'll be shown a conflict resolution screen (see below). New projects with no conflicts are added automatically. Catalog/crew/misc settings in the file are ignored in merge mode — only projects are merged.
  • Import & Replace All — wipes everything on this device and replaces it with the contents of the file (projects, plus catalog/crew/misc if present in the file). Use this for restoring a full backup onto a fresh install, or intentionally syncing one device to match another exactly.
Resolving Merge Conflicts

When "Import & Merge" finds a project ID that already exists on this device, you'll see each conflicting project with three choices:

  • Keep my version — ignore the imported copy, no change (default).
  • Use imported version — overwrite the local project with the imported one.
  • Keep both — leaves your version as-is and adds the imported copy as a new project (renamed with "(imported)" and given a new ID).

Click "Apply & Merge" once you've chosen for each conflict. Projects with no conflicts are added regardless.

Cross-Device Workflow

For syncing individual jobs between devices (e.g. Mac ↔ iPad): check the project(s) you've been working on, "Export Selected," transfer the file (AirDrop, email, Drive), then on the other device use "Import & Merge." If both devices edited the same project since the last sync, you'll be prompted to choose which version to keep.

Auto-Backups

The app also keeps automatic local snapshots in the background (every few minutes while in use, up to the last 8). Click ↻ Auto-backups (in the Sync menu) to view, download, or restore one of these snapshots — useful if you accidentally delete or overwrite a project. Auto-backups live in the same browser storage as your data, so they don't protect against clearing the browser cache; for that, use a downloaded export file.

Reset to New-User State

⚠ Reset to New-User State (bottom of the Sync menu) permanently erases all projects, catalog, crew rates, misc items, setup status, and auto-backups on this device, then reloads — simulating exactly what a brand-new install looks like. Requires two confirmations. Export Everything (Projects + Settings) first if you want to restore afterward — this cannot be undone and is not itself covered by auto-backup (auto-backups are cleared as part of the reset).

After a reset, you'll still see a starter list of turf, infill, and rock products (names only, no vendor pricing) and a starter labor rate / price sheet list (item names and units, but every rate field is blank). This is intentional — a brand-new install ships with these starter lists so you're not starting from a completely empty catalog, but with zero pricing pre-filled so nothing gets quoted using placeholder numbers. Fill in your real costs and rates in Settings before creating quotes.

Recommended Workflow
  1. Set up your Settings once (turf products, infill, rock, misc items, crew rates, profit margin)
  2. Create a new project for each customer job — optionally import a Moasure CSV right away
  3. On the Quote Builder tab, enter your turf rows — use Base Yard for the standard product, Alt Turf Option for any upgrade options, and Putting Green if applicable
  4. Infill auto-populates from turf sqft — assign tiers (Standard / Upgraded) to generate infill options in quotes
  5. Enter edging linear footage
  6. Add any misc items needed for the job
  7. On the Layout tab, set roll direction/width and review the roll plan. If the Moasure file has a separate putting green outline, mark that layer as Putting Green and configure the Putting Green Fringe (product + width) if this job needs one
  8. Check the Piece List in Roll Results — this is your cut sheet for installers, with length/width per piece and total linear footage
  9. Review the Quote Options section — all combinations are generated automatically, including fringe cost where applicable
  10. Switch to the Materials tab to see ordering quantities
  11. Use the quote totals to build your Jobber quote
